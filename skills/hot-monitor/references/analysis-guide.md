# 内容分析指南

当 Hot Monitor CLI 返回原始采集结果后，由 AI Agent 完成内容分析。本指南提供标准化的分析框架。

## 分析流程

```
原始数据 → 相关性判断 → 虚假识别 → 来源加权 → 综合评分 → 输出报告
```

## 1. 相关性判断

对每条结果，判断其与监控关键词的关联程度：

| 等级 | 标识 | 定义 | 示例 |
|------|------|------|------|
| **完全匹配** | `exact_match` | 标题/摘要直接讨论目标关键词 | 搜"GPT-5"，标题"GPT-5 正式发布" |
| **部分匹配** | `partial_match` | 内容部分涉及，但不是核心主题 | 搜"GPT-5"，标题"AI 模型对比：从 GPT-4 到 Claude" |
| **间接关联** | `tangential` | 仅背景信息涉及或同类话题 | 搜"GPT-5"，标题"OpenAI 发布新 API 定价策略" |
| **不相关** | `unrelated` | 搜索噪声，完全无关 | 搜"GPT-5"，标题"GPT 磁盘分区工具使用教程" |

## 2. 虚假/低质内容识别

以下特征应降低评分，甚至直接标记为虚假：

### 🚩 高风险信号
- 标题包含夸张词汇（"震惊"、"重磅"、"突发"、"全网疯传"）但摘要空洞
- 标题与摘要严重不符（标题党）
- URL 来自已知内容农场域名
- 无明确发布时间或发布在未来
- 与知名事件同名但内容为广告
- 纯 AI 生成痕迹明显（重复句式、无实质信息）

### 🟡 中风险信号
- 来源为个人博客/Twitter/论坛且无引用来源
- 摘要仅一句话或无摘要
- 域名看起来像仿冒（如 `techcrunch.co` vs `techcrunch.com`）

## 3. 来源可信度评估

| 等级 | 类型 | 典型域名 | 建议权重 |
|:----:|------|---------|:-------:|
| T1 | 官方/权威机构 | gov.cn, xinhuanet.com, reuters.com, bbc.com | 0.98 |
| T2 | 知名科技媒体 | techcrunch.com, theverge.com, arstechnica.com, 36kr.com | 0.96 |
| T3 | 知名技术社区 | github.com, zhihu.com, juejin.cn, medium.com | 0.94 |
| T4 | 社交媒体 | twitter.com/x.com, reddit.com, bilibili.com, weibo.com | 0.92 |
| T5 | 未知/其他 | 无法归类 | 0.90 |

## 4. 综合评分

建议公式：

```
综合分 = 相关性(0-1) × 来源权重 + 时效性加分 + 信息量加分
```

- **相关性**: `exact_match`=1.0, `partial_match`=0.7, `tangential`=0.4, `unrelated`=0.0
- **来源权重**: 按上表
- **时效性加分**: 24h 内 +0.05, 7 天内 0, 30 天外 -0.05
- **信息量加分**: 有详细摘要 +0.03, 仅标题 -0.02

最终评分上限 1.0。

## 5. 阈值建议

| 阈值 | 处理方式 |
|:----:|---------|
| ≥ 0.7 | 🔥 高相关热点 — 重点呈现 |
| 0.4 ~ 0.7 | 📰 一般提及 — 简要列出 |
| < 0.4 | 🛡️ 不展示或标注为"低相关" |
| 明显虚假 | ❌ 直接过滤，单独列出警告 |

## 6. 输出模板

分析完成后，用以下 Markdown 格式向用户报告：

```markdown
## 🔍 "[关键词]" 热点分析

**扫描时间:** 2026-05-22 14:30 | **来源数:** 7 | **采集数:** 45 | **有效数:** 12

### 🔥 高相关热点
| # | 标题 | 来源 | 得分 | 关联度 | 摘要 |
|---|------|------|:----:|:------:|------|
| 1 | GPT-5 正式发布... | TechCrunch | 0.95 | 完全匹配 | OpenAI 发布... |
| 2 | ... | ... | ... | ... | ... |

### 📰 一般提及
- **标题1**（来源，0.55 分）— 部分匹配，涉及但非核心主题
- ...

### 🛡️ 已过滤
- **标题X** — 原因：标题党，内容为广告
- **标题Y** — 原因：不相关（GPT 磁盘工具）
```

## 7. 特殊情况处理

### 结果过多（>30 条）
优先展示：高相关度 + 高来源可信度 + 近 7 天发布
折叠其余为"查看更多"

### 结果过少（<3 条）
提示用户：
- 关键词可能太窄，建议放宽或添加同义词
- 可尝试切换信息源组合
- 检查 `check` 命令确认源可用性

### 全是低质结果
如实告知用户，并建议：
- 更换关键词
- 增加信息源
- 手动降低阈值
